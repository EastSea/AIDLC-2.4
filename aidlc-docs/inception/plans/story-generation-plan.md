### Question 1: 스토리 분류 방식
User Stories를 어떤 기준으로 분류할까요?

A) 사용자 유형별 (고객용 스토리 / 관리자용 스토리)
B) 기능 영역별 (메뉴, 장바구니, 주문, 테이블 관리 등)
C) 사용자 여정별 (주문 플로우, 관리 플로우)
D) 기타 (아래 [Answer]: 태그 뒤에 설명해 주세요)

[Answer]: B

---

### Question 2: 스토리 상세 수준
각 User Story의 상세 수준은 어느 정도로 할까요?

A) 간결하게 (핵심 기능 위주, 스토리당 1-2개 Acceptance Criteria)
B) 상세하게 (세부 시나리오 포함, 스토리당 3-5개 Acceptance Criteria)
C) 기타 (아래 [Answer]: 태그 뒤에 설명해 주세요)

[Answer]: B

---

### Question 3: 페르소나 상세 수준
사용자 페르소나를 얼마나 상세하게 정의할까요?

A) 기본 (이름, 역할, 주요 목표만)
B) 상세 (이름, 역할, 목표, 특성, Pain Points, 사용 시나리오)
C) 기타 (아래 [Answer]: 태그 뒤에 설명해 주세요)

[Answer]: B

---

### Question 4: 직원호출 기능 범위
UI 레이아웃에 "직원호출" 버튼이 포함되어 있는데, 이 기능의 범위는 어떻게 할까요?

A) 단순 알림 (관리자 화면에 호출 알림만 표시)
B) 호출 + 응답 (관리자가 응답 가능, 고객에게 응답 표시)
C) 이번 MVP에서 제외 (버튼만 표시, 기능 미구현)
D) 기타 (아래 [Answer]: 태그 뒤에 설명해 주세요)

[Answer]: A

---

### Question 5: 인원수 필터 기능
메뉴에 인원수 필터(1~2인, 3~4인 등)가 있는데, 이 정보는 어떻게 관리할까요?

A) 메뉴 등록 시 인원수 태그 추가 (관리자가 설정)
B) 메뉴 카테고리로 관리 (1~2인 메뉴 카테고리, 3~4인 메뉴 카테고리)
C) 이번 MVP에서 제외
D) 기타 (아래 [Answer]: 태그 뒤에 설명해 주세요)

[Answer]: A
