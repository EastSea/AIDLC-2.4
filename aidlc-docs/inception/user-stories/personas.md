# User Personas

## Persona 1: 고객 (Customer)

| 항목 | 내용 |
|------|------|
| **이름** | 김민수 (대표 고객) |
| **역할** | 식당 방문 고객 |
| **연령대** | 20~50대 |
| **기술 숙련도** | 다양함 (스마트폰 사용 가능 수준) |

### 주요 목표
- 빠르고 편리하게 음식 주문
- 직원 호출 없이 자율적으로 주문
- 주문 상태 확인

### 특성
- 터치 기반 인터페이스에 익숙
- 대기 시간에 민감
- 직관적인 UI 선호

### Pain Points
- 직원을 기다리는 시간이 길다
- 메뉴판이 복잡하거나 가격이 안 보인다
- 주문이 제대로 들어갔는지 불안하다

### 사용 시나리오
1. 테이블에 앉아 태블릿/키오스크 화면 확인
2. 카테고리별로 메뉴 탐색
3. 원하는 메뉴를 장바구니에 담기
4. 주문 확정 후 주문 번호 확인
5. 필요시 추가 주문 또는 직원 호출

## Persona 2: 매장 관리자 (Admin)

| 항목 | 내용 |
|------|------|
| **이름** | 박서연 (매장 매니저) |
| **역할** | 매장 운영 관리자 |
| **연령대** | 30~50대 |
| **기술 숙련도** | 중간 (기본 PC/태블릿 사용 가능) |

### 주요 목표
- 실시간으로 주문 현황 파악
- 효율적인 테이블 관리
- 메뉴 정보 관리

### 특성
- 바쁜 환경에서 빠른 판단 필요
- 여러 테이블 동시 관리
- 정확한 정보 필요

### Pain Points
- 주문이 밀리면 혼란스럽다
- 테이블별 주문 현황 파악이 어렵다
- 메뉴 변경 시 수작업이 많다

### 사용 시나리오
1. 매장 오픈 시 관리자 로그인
2. 대시보드에서 실시간 주문 모니터링
3. 주문 상태 변경 (대기중 → 준비중 → 완료)
4. 테이블 이용 완료 처리
5. 필요시 메뉴 정보 수정
