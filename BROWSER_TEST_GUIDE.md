## 🧪 테스트 시나리오

### 시나리오 1: 고객 - 메뉴 브라우징 및 주문

#### Step 1: 테이블 로그인
1. 브라우저에서 `http://localhost:3000/login` 접속
2. 다음 정보 입력:
   - 매장 코드: `STORE01`
   - 테이블 번호: `5`
   - 비밀번호: `1234`
3. **로그인** 버튼 클릭
4. ✅ 메뉴 페이지로 이동 확인

#### Step 2: 메뉴 브라우징
1. 메인 페이지 (`http://localhost:3000`)에서 메뉴 목록 확인
2. 카테고리 필터 테스트:
   - "전체" 버튼 클릭
   - 각 카테고리 버튼 클릭하여 필터링 확인
3. 인원수 필터 테스트:
   - 드롭다운에서 "1인분", "2인분" 등 선택
   - 필터링된 메뉴 확인

#### Step 3: 장바구니 담기
1. 메뉴 카드에서 **장바구니 담기** 버튼 클릭
2. 여러 메뉴 추가
3. 브라우저 주소창에 `/cart` 입력하여 장바구니 페이지 이동
4. ✅ 담은 메뉴들이 표시되는지 확인

#### Step 4: 장바구니 관리
1. 수량 증가/감소 버튼 (+/-) 테스트
2. **삭제** 버튼으로 아이템 제거 테스트
3. 총 금액 계산 확인
4. ✅ localStorage 테스트: 페이지 새로고침 후 장바구니 유지 확인

#### Step 5: 주문하기
1. **주문하기** 버튼 클릭
2. ✅ "주문이 완료되었습니다" 알림 확인
3. 자동으로 주문 내역 페이지로 이동 확인

#### Step 6: 주문 내역 확인
1. 주문 내역 페이지에서 방금 주문한 내역 확인
2. 주문 번호, 메뉴, 금액, 상태 확인
3. ✅ 주문 상태가 "대기중"으로 표시되는지 확인

---

### 시나리오 2: 관리자 - 주문 관리

#### Step 1: 관리자 로그인
1. 브라우저에서 `http://localhost:3000/admin/login` 접속
2. 다음 정보 입력:
   - 매장 코드: `STORE01`
   - 사용자명: `admin`
   - 비밀번호: `admin123`
3. **로그인** 버튼 클릭
4. ✅ 관리자 대시보드로 이동 확인

#### Step 2: 주문 관리
1. 대시보드에서 주문 목록 확인
2. 주문 카드에서 상태 드롭다운 변경:
   - "대기중" → "조리중" 변경
   - "조리중" → "완료" 변경
3. ✅ 상태 변경이 즉시 반영되는지 확인

#### Step 3: 주문 삭제
1. 주문 카드에서 **삭제** 버튼 클릭
2. 확인 다이얼로그에서 "확인" 클릭
3. ✅ 주문이 목록에서 사라지는지 확인

---

### 시나리오 3: 관리자 - 메뉴 관리

#### Step 1: 메뉴 관리 페이지 접속
1. `http://localhost:3000/admin/menus` 접속
2. ✅ 메뉴 목록이 표시되는지 확인

#### Step 2: 새 메뉴 추가
1. **새 메뉴 추가** 버튼 클릭
2. 폼에 다음 정보 입력:
   - 카테고리 ID: `cat1`
   - 메뉴명: `테스트 메뉴`
   - 가격: `15000`
   - 설명: `테스트용 메뉴입니다`
   - 인원: `2인분`
3. **저장** 버튼 클릭
4. ✅ 메뉴 목록에 새 메뉴가 추가되는지 확인

#### Step 3: 메뉴 수정
1. 메뉴 항목에서 **수정** 버튼 클릭
2. 가격을 `18000`으로 변경
3. **저장** 버튼 클릭
4. ✅ 변경사항이 반영되는지 확인

#### Step 4: 메뉴 삭제
1. 메뉴 항목에서 **삭제** 버튼 클릭
2. 확인 다이얼로그에서 "확인" 클릭
3. ✅ 메뉴가 목록에서 사라지는지 확인

## 🔍 추가 테스트 항목

### localStorage 테스트
1. 장바구니에 아이템 추가
2. 브라우저 개발자 도구 열기 (F12)
3. Application → Local Storage → `http://localhost:3000` 확인
4. `cart` 키에 장바구니 데이터 저장 확인
5. 페이지 새로고침 후 장바구니 유지 확인

### 반응형 디자인 테스트
1. 브라우저 개발자 도구 열기 (F12)
2. 디바이스 툴바 토글 (Ctrl+Shift+M)
3. 다양한 화면 크기에서 레이아웃 확인:
   - Mobile (375px)
   - Tablet (768px)
   - Desktop (1920px)

### 에러 처리 테스트
1. Mock API 서버 중지
2. 로그인 시도
3. ✅ 에러 메시지가 표시되는지 확인
4. Mock API 서버 재시작
5. 정상 동작 확인

## 🐛 문제 해결

### Mock API 서버가 시작되지 않을 때
```bash
# Python 버전 확인
python3 --version

# 필요한 경우 Python 설치
# macOS: brew install python3
```

### Frontend 서버가 시작되지 않을 때
```bash
# 의존성 재설치
cd web
rm -rf node_modules package-lock.json
npm install
npm start
```

### 포트가 이미 사용 중일 때
```bash
# 8001 포트 사용 프로세스 확인
lsof -i :8001

# 3000 포트 사용 프로세스 확인
lsof -i :3000

# 프로세스 종료 (PID는 위 명령어 결과에서 확인)
kill -9 <PID>
```
